# Figma UI Companion System Map

This file maps the Companion UI for the Figma return.

## Public room language

Use these public labels in Figma unless there is a specific reason not to:
- `CUES`
- `ACTS`
- `PLAY`
- `TALK`
- `TUNE`
- `KNOW`
- `ECHO`

## Internal alias notes

Code still contains some transition naming.
Use this only as an implementation bridge.

- `home` in code maps to the Companion home glass and should be treated as `CUES`
- `flow` in code maps to `ACTS`
- `sync`/children in code are part of the `KNOW` family direction
- `echo` maps to `ECHO`
- `plot`, `map`, `link` remain useful child labels under `ECHO`

## Shell law

The Companion should continue to behave as one organism:
- `Surface` = present room
- `Anchor` = invocation hand
- `Stream` = beneath / journey layer

Do not design this as tab-bar software.
Do not fragment it into page templates.

## Companion families

### `CUES`
The home bloodline of the app.
This should still feel alive, flowing, and bodily.

### `ACTS`
The beneath / stream layer.
Longer-form continuity and lived movement over time.

### `TALK`
The listening corridor.
Quiet, held, dark, exact.

### `PLAY`
The frequency room.
Sonic, adaptive, atmospheric.

### `TUNE`
The body regulation family.
Practice, rhythm, physical shift.

### `KNOW`
The understanding family.
Reading, insight, reflection, and explanatory depth.

### `ECHO`
The proof/tether/listening family.
Telemetry, integration, support, infrastructure, and reflection of the wider system.

## Highest-priority child rooms right now

### `READ`
The Infinite Book.
One pristine reading room.
No search UX.
No visible library mechanics.

### `PLOT`
Three bodily telemetry columns.
A check-in, not a questionnaire.

### `MAP`
A living constellation.
Spatial understanding, not analytics.

### `LINK`
The tether layer.
Support, contacts, integrations, and export.
Should feel infrastructural without losing warmth.

## Returnability test

Every Companion frame should still answer:
- would someone come back to this tomorrow?
- does this reduce burden?
- does this feel like software or like a place?
