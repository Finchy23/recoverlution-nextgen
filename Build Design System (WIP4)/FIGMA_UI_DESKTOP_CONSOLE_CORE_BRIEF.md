# Figma UI Desktop Console/Core Brief

This brief exists so the next Figma pass can cleanly split Companion from the operational layers.

## Product split

### Companion
App-first.
Private.
Bodily.
Returnable.

### Professional Console
Desktop-first.
Practitioner workbench.
Focused on day-to-day operational flow.

### Organisational Core
Desktop-first.
Administrative and organisational control layer.
Focused on policy, oversight, utilisation, and system health.

## Professional Console should hold
- today's schedule
- upcoming sessions
- client/patient roster
- quick profile detail
- booking links
- availability controls
- video/join links
- payment issues and exceptions
- session prep/follow-up entry points

## Organisational Core should hold
- team members and roles
- org scheduling policy
- provider connection health
- revenue and utilisation
- audit posture
- billing posture
- org-wide configuration
- exceptions and alerts

## Desktop UI principle

Do not make these feel like larger mobile rooms.

Design them as workbenches:
- clear navigation
- dense but elegant information
- confident hierarchy
- fast scanning
- deliberate detail panes
- obvious operational actions

## Shared with Companion
These may share brand DNA:
- typography discipline
- restraint in color
- trust posture
- calm surfaces
- high-quality motion where helpful

## Not shared with Companion
These should diverge:
- density
- panel complexity
- table usage
- multi-step workflows
- dashboards
- admin posture

## Lightweight mirror back into Companion
Only bring these back into Companion:
- next session
- quick join
- urgent booking change
- support or SOS handoff
- simple payment issue requiring action
- simple availability state

Keep the rest in desktop.

## First Figma deliverables
1. Professional Console top-level IA
2. Organisational Core top-level IA
3. key home/dashboard views for each
4. one detail view for practitioner/client workflow
5. one operational health/dashboard view for Core
