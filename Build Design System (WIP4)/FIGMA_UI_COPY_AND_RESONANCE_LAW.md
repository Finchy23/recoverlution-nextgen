# Figma UI Copy and Resonance Law

This file captures the copy and tone law that Figma should design against.

## Core principle
The system is a catalyst, not a cure.
It should feel like a tuning fork, not a diagnosis engine.

## What to avoid
- assumption-heavy app-time language
- "the engine is cold" style phrases
- telling the user what they feel
- congratulating the user too neatly
- overclaiming the shift
- generic product helper copy
- wellness-app encouragement language

## Entrance law
Entrances should:
- widen the aperture
- open a perspective
- invite, not diagnose
- feel true whether the person arrived 10 seconds ago or 20 minutes ago

## Kinetic law
When the user touches the glass:
- words should step back
- physics should lead
- the body should do the interpretation
- copy should become material, not commentary

## Seal law
The end of a scene should:
- leave space
- never overclaim outcome
- avoid software finality
- feel like a lingering chord, not a green checkmark

## READ law
`READ` should feel timeless and exact.
Never let it feel like article content marketing.

## ECHO law
`ECHO` should feel infrastructural, biological, and careful.
Never let consent or integration moments feel like corporate setup screens.

## Companion-wide standard
The right question is not:
- is this polished?

The right questions are:
- does this resonate?
- does this respect uncertainty?
- does this feel like a place the user can come home to?
