# Recoverlution Creative Brief: Controls, Navigation, and Feel

Owner: `@frontend`  
Date: `2026-03-10`  
State: `review draft`

## Purpose

This is a creative-facing brief.

It is intentionally not written from the code outward.

Its job is to help a designer or creative approach the product with a fresh eye while still understanding:
- what the controls are really doing
- how surfaces should transform
- how navigation should feel
- what makes the glass magical
- what would break the spell

This is not a wireframe brief.
It is a behavioral and emotional brief.

---

## 1. The core idea

The product should not feel like moving around an app.

It should feel like staying still while the environment changes around you.

The user does not “go to another page.”
The glass:
- gathers
- opens
- deepens
- routes
- seals
- remembers

The experience should feel less like navigation and more like:
- invocation
- descent
- release
- return

If a user feels like they are managing software, the spell is broken.

---

## 2. The shell the user lives inside

There are only three permanent elements in the system:

### The Surface
The main field.
This is where the room actually happens.

It should feel:
- edge-to-edge
- dominant
- immersive
- materially alive

It should not feel:
- like a page container
- like a viewport into another app
- like a card on a background

### The Anchor
The glowing point at the base of the glass.

This is not just a control.
It is the user’s tether to the OS.

It should feel:
- grounding
- ritual
- bodily
- weighted

It should not feel:
- like a tab bar
- like a mode switch
- like a menu

### The Stream
The horizon at the top.

This is not a notification rail.
It is an ambient current.

It should feel:
- peripheral
- flowing
- weather-like
- calm

It should not feel:
- like alerts
- like admin
- like productivity software

---

## 3. What the controls should feel like

We are not trying to make “beautiful buttons.”
We are trying to make the user feel that the glass responds to intention.

The control language should come from:
- pressure
- gravity
- pull
- drift
- resistance
- yield
- release

Not from:
- menus
- toggles
- dropdowns
- obvious CTA stacks

### Press
Use when something is immediate and clean.

Should feel like:
- touching the current
- entering the room
- accepting what is already ready

### Hold
Use when the moment requires intention.

Should feel like:
- grounding
- commitment
- asking the system to go deeper

This is one of the most magical gestures in the whole OS.

### Drag
Use when the body needs to participate.

Should feel like:
- moving through matter
- redirecting force
- creating change physically

### Scroll / Descent
Use for depth, not browsing.

Should feel like:
- going further in
- not consuming more content

### Tap on a live element
Use when the field already makes the target obvious.

Should feel like:
- touching a live node
- following a current
- activating a relationship already present in the room

---

## 4. How movement between surfaces should work

All transitions should feel like one material recomposing.

There are only a few true movement types:

### Invocation
A room is summoned because the moment asks for it.

This should feel like:
- the environment preparing itself
- a field gathering
- the right state arriving

Not:
- opening a module
- starting a tool

### Deepening
The user stays in the same room, but the room opens further.

This is the ideal mode for:
- Journeys
- Insights
- Read

### Handoff
One room routes into another without breaking the spell.

The user should feel:
- the current continued

Not:
- a feature switch happened

### Continuation
The room remembers and reopens.

This should feel:
- held
- resumed
- already alive

Not:
- resume session
- continue where you left off

### Seal
The room leaves behind a trace.

This is critical.
Without a seal, the room feels decorative.

---

## 5. The emotional sequence every room should obey

Every meaningful room should move through some version of:

1. Arrival  
The atmosphere gets there first.

2. Threshold  
The user crosses from passive into engaged.

3. Live State  
The room is fully present. Chrome should recede.

4. Shift  
Something changes because the user acted, held, or stayed.

5. Seal & Carry  
The moment leaves behind a trace in the shell, stream, graph, or route.

This sequence matters more than any one visual treatment.

---

## 6. The feeling of each room family

### Callable Rooms
`NaviCues`, `Talk`, `Play`, `Studio`

These should feel:
- summonable
- immediate
- low-ceremony
- biologically useful

The user should feel:
- I can enter this now
- this knows what the moment needs

### Slow-Depth Rooms
`Journeys`, `Insights`, `Read`, `Practices`

These should feel:
- spacious
- inevitable
- depth-led
- not content-shaped

The user should feel:
- I am going further in
- not: I am consuming another thing

### Intelligence Rooms
`Signal`, `Navigate`

These should feel:
- believable
- relational
- structurally intelligent
- quietly powerful

The user should feel:
- the system knows what matters
- support is near
- change is visible

Not:
- this is an analytics layer
- this is workflow software

---

## 7. What makes the glass magical

The magic is not visual excess.

The magic is:
- atmosphere arriving before words
- the shell saying very little
- controls appearing late
- pressure feeling physical
- routes being implied before they are named
- proof becoming visible through shape and hierarchy
- support becoming visible through proximity and rank
- every room leaving a subtle memory behind

The glass should feel:
- aware
- patient
- weighted
- responsive
- quiet

The strongest state is when the user cannot quite tell whether the interface is guiding them or simply revealing what was already true.

---

## 8. What would break the spell

The following are likely failures:

- anything that feels like premium SaaS
- card stacks where one field should exist
- controls that explain themselves too early
- shell copy that repeats what the room already proves
- obvious app navigation behavior
- transport-heavy media controls
- dashboard language in `Signal`
- orchestration/workflow language in `Navigate`
- branching UI in `Talk`
- article UI in `Insights`
- toolkit UI in `Practices`
- editorial clutter in `Read`

If the product starts feeling “well designed” instead of “quietly alive,” it has gone off course.

---

## 9. Questions a creative should work against

Use these questions while designing:

- If I removed half the visible controls, would the room feel weaker or more inevitable?
- Is this room being explained, or is it being felt?
- Does the shell carry too much language?
- Does the user feel invoked, or operated?
- Does the room feel like the same piece of glass as the last room?
- Is the next move already visible in the hierarchy before it is named?
- Does the room leave behind a trace after completion?
- Does this feel like software, or like a held state?

---

## 10. The creative north star

Design currents, not screens.

Design thresholds, not pages.

Design pressure, not buttons.

Design memory, not breadcrumbs.

Design a nervous system, not a product suite.

If done right, the user should feel:
- this is calm
- this is different
- this understands the moment
- this is carrying me

That is the standard.
