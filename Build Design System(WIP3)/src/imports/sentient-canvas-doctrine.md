# Frontend Sentient Canvas Doctrine
Owner: `@frontend`
Date: `2026-03-10`
State: `review draft`
Purpose:
- give design, frontend, copy, architect, and QA one reviewable frontend source of truth for the magic layer of the product
- compress the strongest thinking from shell work, room work, room audits, player governance, copy posture, and runtime constraints into one usable doctrine note
- describe what the browser should feel like without hardening backend-only truth into frontend doctrine
Boundary:
- this is a frontend doctrine candidate
- it defines rendered truth, interaction truth, and composition truth on the glass
- it does **not** finalize backend persistence, auth/trust, or asset-mastering doctrine alone
---
## 1. The landing statement
Recoverlution should feel like one living operating system, not a set of beautiful surfaces.
The browser should behave like:
- one material
- one shell
- one nervous system
- many rooms
- very few controls
- deeply differentiated emotional registers
The user should feel:
- invoked
- carried
- held
- resumed
- changed
The user should not feel:
- switched
- launched
- navigated
- briefed
- managed
The system is strongest when the user cannot quite tell where interface ends and guidance begins.
### 1.1 What frontend should believe
This browser should not feel like software with better styling.
It should feel like:
- a held environment
- a nervous system with memory
- a piece of glass that is aware of urgency, weight, and recovery
- a product that gets quieter as it gets more powerful
The magic is not blur, glow, or motion in isolation.
The magic is:
- atmosphere before explanation
- pressure before buttons
- geometry before labels
- route before instruction
- proof before analytics
- support before orchestration
- memory before navigation
If a surface becomes more expressive but less coherent, it is wrong.
If a surface becomes more consistent but less alive, it is also wrong.
### 1.2 What a UI agent must not do
Do not treat the system as:
- a menu of effects
- a pile of premium components
- a set of room skins
- a collection of beautiful cards
- a content experience with atmosphere added later
Do not “design a screen”.
Design:
- a current
- a threshold
- a field
- a seal
- a remembered trace

## 2. The universal laws
These are the non-negotiable laws that govern every surface.
### 2.1 The Law of Attentional Physics
- the background does not disappear when a room becomes important
- it yields
- blur, darkness, motion reduction, and depth attenuation must protect focus without killing the organism
### 2.2 The Total Energy Equation
- the screen has one energy budget
- if the hero payload needs 90% of attention, the atmosphere must fall back to 10%
- if the room is ambient or resting, the atmosphere can carry most of the emotional force
### 2.3 The Law of Late Arrival
- the most important controls should appear late
- if the room can carry the user through atmosphere, geometry, or gravity alone, the shell should wait
### 2.4 The Law of Evaporation
- when a room becomes active, non-essential chrome must evaporate
- the user should be left inside the experience, not surrounded by explanatory UI
### 2.5 The Five-Step Transition Sequence
Every meaningful surface change should move through:
1. Arrival
2. Threshold
3. Live State
4. Shift
5. Seal & Carry
Nothing should feel like a cut to a new page.
---
## 3. The shell
The shell is the permanent operating frame of the system.
It has only three real pieces:
- `Surface`
- `Anchor`
- `Stream`
### 3.1 Surface
- the immersive hero canvas
- the stage where the room happens
- should feel edge-to-edge and materially dominant
- should not feel like a component wrapper
### 3.2 Anchor
- the ritual point at the base of the glass
- should feel like grounding and invocation, not a control dock
- should carry pressure, not menu logic
### 3.3 Stream
- the horizon
- carries ambient context, continuity, support drift, proof drift, and timing
- should feel like weather, not notifications
### 3.4 Shell ownership
The shell should own:
- back posture
- canopy posture
- threshold posture
- route invocation posture
- continuity language
- reopen behavior
- ambient horizon behavior
Rooms should not keep cloning these.

## 4. The five runtime governor layers
This is the clean runtime framing for the sentient canvas.
### 4.1 Atmosphere
Atmosphere is the spatial and emotional safety of the room.
It governs:
- depth
- shadow
- volumetric feel
- void density
- light placement
- glass thickness
Canonical atmosphere families:
- Sanctuary
- Cryo-Chamber
- Abyssal Void
- Bioluminescent Mesh
- Chiaroscuro Spotlight
- Twilight Shroud
Atmosphere answers:
- how safe does the room feel?
- how open or contained is the void?
- where does the light live?
### 4.2 Motion
Motion is not decoration. Motion is biological rhythm and material behavior.
It governs:
- pacing
- viscosity
- settle curve
- blur/recede behavior
- field responsiveness
Canonical motion families:
- Somatic Breath
- Viscous Unspooling
- Pendulum Settle
- Defocus Dissolve
- Cymatic Ripple
- Tectonic Drift
Motion answers:
- how does the environment breathe?
- how heavy is the digital matter?
- how does tension leave?
### 4.3 Interaction
Interaction is the sentient response of the environment.
It governs:
- parallax
- haptic synchrony
- displacement
- refraction
- voice-reactive response
- resistance
Canonical interaction families:
- Gyroscopic Parallax
- Haptic Entrainment
- Floating Particle Displacement
- Surface Refraction
- Acoustic Flutter
- Viscous Drag
Interaction answers:
- how does the glass know I’m here?
- how does the room respond to touch, voice, and pressure?
### 4.4 Temperature
Temperature is not a style. It is a governor.
It governs:
- cognitive allowance
- urgency
- motion permission
- copy allowance
- whether the room should narrow or open
Canonical heat bands:
- Band 4: Survival Mode
- Band 3: Sympathetic Activation
- Band 2: Early Dysregulation
- Band 1: Alert & Regulated
- Band 0: Safe & Social
Temperature answers:
- how much can the room ask of the user right now?
- how fast or slow may the system become?
### 4.5 Color
Color is not branding. It is psychological frequency.
It governs:
- emotional wavelength
- intervention tone
- visible nervous-system priming
- proof and carry signatures
Canonical color signatures:
- Neural Reset
- Amber Resonance
- Quiet Authority
- Sacred Ordinary
- Verdant Calm
- Void Presence
- Twilight Shift
Color answers:
- what kind of emotional or biological signal is this room projecting?

## 5. The meaning of “magic”
Magic in this system does not mean ornament.
Magic means:
- the room feels alive before the user understands it
- the controls feel like pressure rather than UI
- the atmosphere feels aware of context
- the shell remembers the current
- the room grows simpler as engagement deepens
- proof is seen in field behavior, not explained through software language
In practical terms, magic is created when:
- atmosphere leads
- copy arrives late
- controls appear only when necessary
- geometry, rank, spacing, and motion do more than labels
- the room leaves behind a visible trace
### 5.1 What a magical room actually does
A magical room does not just look beautiful.
It:
- lowers the effort of entering
- reduces the number of decisions the user must make
- teaches the body what to do through the field itself
- changes state without needing a paragraph to justify itself
- leaves behind evidence that the moment mattered
That means a room should usually become:
- quieter under engagement
- clearer under pressure
- simpler after commitment
- more personal after completion
### 5.2 How to compose a room
A UI agent should compose in this order:
1. What is the user's likely state?
2. What is the room's therapeutic job?
3. What should the shell own here?
4. What should the atmosphere do before copy appears?
5. What is the primary artifact or field behavior?
6. What is the minimum viable copy?
7. What trace or route should remain afterward?
If the answer starts with “which visual treatment should I use?”, the order is already wrong.

## 6. The control vocabulary
The product should use a very small number of control primitives.
### Press
Use for:
- invocation
- enter
- continue
- activate
Feel:
- immediate
- clean
### Hold
Use for:
- deep invocation
- state shift
- grounding
- escalation
Feel:
- weighted
- intentional
- embodied
### Drag
Use for:
- somatic mechanics
- corridor progression
- field exploration
- directional release
Feel:
- physical
- inevitable
### Scroll / Descent
Use for:
- deepening
- not browsing
Feel:
- entering depth
### Tap on live element
Use for:
- selecting a focus zone
- following a route
- touching a current
Feel:
- direct contact with the room
### 6.1 What makes a control magical
A magical control:
- appears late
- feels physical
- says almost nothing
- inherits the room's emotional law
- leaves a trace when it matters
A weak control:
- explains itself too early
- duplicates what the field already proves
- looks selectable before it feels inevitable
- survives too long once the room becomes active

## 7. The copy system
Copy should regulate, orient, guide, and seal.
It should say less and arrive late.
### Copy layers
- Eyebrow
- Canopy
- Guide
- Gesture
- Receipt
- Route
- Ambient
### Rule of use
- if the field already says it, delete the text
- if the shell is explaining what the room already proves, delete the shell line
- if the room can orient with one line, do not use three
### Copy density
- whisper
- light
- moderate
### Copy ownership
Player-owned:
- receipt posture
- route posture
- continuity posture
- ambient drift posture
Room-owned:
- canopy
- gesture
- domain-specific guide
- room-specific emotional register

- benchmark room
- strongest authored entrance / mechanic / seal relationship
- minimal copy in live state

### Talk
- a corridor, not chat
- openings should feel carried, not branched
- route should emerge naturally from the room
### Play
- a sentient mix, not a library

- less transport UI, more sonic field
### Studio
- sanctuary, not streaming app
- poster-to-immersion
- almost no media chrome
### Journeys
- path over time
- minimal scene bookkeeping
- real-world return matters
### Insights
- descent, not article
- documentary depth fused with somatic lock points
### Read

### Practices
- instruments, not toolkit
- held execution shell

### Navigate

## 10. Attentional physics
This is one of the most important design laws in the system.
### Dynamic Depth of Field
- when the user needs focus, the background yields through blur, darkness, and depth recession
- the environment should not die
- it should defocus
### Kinetic Subordination
- when the foreground needs attention, the background motion must reduce
- the background should not compete with the intervention
### Room-specific attenuation
- Read / Articles: maximum attenuation
- NaviCues: harmonic fusion between background and hero mechanic
- Talk: darkened, listening background with minimal distraction
### Total Energy Equation
- the screen has one energy budget
- background and foreground must trade energy depending on the room and state
### 10.1 The yield model
The environment should never “turn off” unless the safety floor requires it.
It should:
- yield
- defocus
- darken
- slow
- hold
The user should feel the OS protecting focus, not removing beauty.
---
## 11. Sequence thermodynamics
Single interventions are not enough. Sequences matter.
The system must avoid habituation and emotional flatness.
### Law of Cooldown
- a high-heat sequence should not remain at the same intensity
- the room should visibly cool, soften, or integrate over time
### Arc patterns
Sequences should follow structured arcs such as:
- Plateau & Release
- Descent & Return
- Containment to Bloom
### Carry
- one cue or room should leave a particle, trace, or current that becomes the seed of the next state
- no cut-to-black thinking
---
## 12. Failure and resilience law
The system must stay magical when reality breaks.
### Clouded Glass
- failure should feel like environmental dissonance, not modal error UI
### Instinctive Affordance
- the room should teach touch through gravity, light, and current
- not through tutorial overlays
### Sensory Override
- a hard floor must exist for sensory-sensitive users
- the system must be able to collapse to quiet monochrome safety
### Acoustic Materiality
- the OS needs micro-acoustic rules, not just `Play`
- sound should confirm weight, drag, release, and receipt
### Hardware Symbiosis
- the OS must adapt to battery, low-power, and hardware fatigue
- atmosphere and fidelity should reduce gracefully
### 12.1 Anti-fragility principle
When reality breaks, the product should become:
- simpler
- quieter
- clearer
- safer
Not:
- louder
- more literal
- more software-like
A broken state should still feel like Recoverlution.
---
## 13. Semantic physics engine
Words are not labels. They are material.
Typography should behave like part of the room’s physics.
### The four semantic dimensions
- heavy matter
- kinetic text
- atmospheric subtext
- semantic morph
### Vocal threading
The same atom or mechanic can be threaded through different voice lanes:
- Companion
- Activator
- Mirror
- and other governed voice archetypes
### Timing
- text should never arrive at the same instant as room load
- atmosphere first
- language second
### Bounding box
- copy must be constrained by character and line budgets
- the payload must fit the glass, not break it
### 13.1 The semantic standard
The semantic layer should feel like:
- truth arriving at the right time
- not interface copy trying to sound poetic
The line should do one of four things:
- regulate
- orient
- guide
- seal
If it is branding, explanation, or filler in disguise, it should be removed.
---
## 14. What is forbidden
These should be treated as rejection criteria.
- painted containers as default UI
- obvious menu logic in the shell
- generic tab-bar thinking
- visible over-explanation
- stacked shell recap
- copy doing the job of geometry
- harsh cut-to-black exits
- shell intrusion where room physics already communicate the action
- biology-clashing combinations
  - for example calming color with violent motion, or high-heat urgency with expansive contemplative pacing
