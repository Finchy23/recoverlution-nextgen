# Recoverlution Frontend Guidelines

This workspace is a sensory shell and proving ground for the Recoverlution organism.

It is not the authority for runtime behavior, auth truth, analytics truth, or asset delivery doctrine.

## Core rule

- frontend owns the room
- runtime owns the truth

## Current build stance

Treat this workspace as:
- a shell of opportunity
- a place to prove the organism law
- a place to harden interaction and spatial logic before full runtime hookup

Do not treat it as:
- a second backend
- a doctrine-authority replacement
- a final naming authority

## Primary frontend law

The shell should be built around:
- `Surface`
- `Anchor`
- `Stream`

This law matters more than individual room polish.

## What to optimize for

- singularity of the glass
- low-friction returnability
- calm emotional precision
- resilient transitions
- strong invocation and return behavior
- minimal box-thinking

## What frontend should own

- shell hierarchy
- room behavior
- invocation and return law
- yielding and attention law
- fallback and degraded mode behavior
- event hook points
- media shell adapters
- mock/live runtime boundaries

## What frontend must not invent

Do not harden or redefine:
- compile law
- auth and trust rules
- analytics truth
- queue and mutation truth
- support / SOS truth
- semantic and orchestration truth
- media delivery doctrine

If the runtime is not ready, mock it behind a boundary instead of creating alternate truth.

## Required build mode

Everything should support:
1. `mock shell mode`
2. `live-contract mode`
