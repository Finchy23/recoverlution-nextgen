# 248 Public Site Review Doctrine

Purpose: define what counts as real progress when the clean-build public site is reviewed.

## Review rule
A page or family is not stronger because it is newer. It is stronger only if it improves coherence, emotional contact, clarity, and launch truth without increasing cognitive burden.

## Review questions
1. Does this feel like one company, one calm intelligence, one unmistakable surface?
2. Is the page carrying its correct burden, or trying to do someone else’s job?
3. Is the copy consumer-first and restrained?
4. Are the assets and motion serving the page burden, or compensating for weak structure?
5. Is the commercial path calm and honest?
6. Does anything in the experience imply doctrine or product truth that is still unresolved?

## Failure conditions
A route fails review if:
- it inherits the old homepage bias without interrogation
- it reintroduces feature-grid thinking
- it behaves like a dashboard
- it leans on expert-runtime content as the primary message layer
- it hardens unresolved naming/media/auth doctrine
- it creates a second visual language inside the same public family

## Success conditions
A route passes review if:
- it is family-consistent
- it is burden-correct
- it is calmer and more convincing than the route it replaced
- it preserves route spine and system truth
- it can be evidenced against acceptance and family gates
