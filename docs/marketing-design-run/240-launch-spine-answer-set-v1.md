## Purpose
- provide the first working answer set for Launch Spine family design without freezing unresolved doctrine

## Launch Spine truth
- the homepage is an arrival surface
- `/pricing` is a decision surface
- `/enter` is a continuity surface
- all three must feel like one calm intelligence

## Working answers

### What the homepage must do
- establish the category without over-explaining it
- create a biological exhale before it creates product understanding
- prove that the company is a system, not a stack of features
- hand trust-building off to `/product` and `/method`

### What the homepage must not do
- explain every modality
- carry the full proof burden
- feel like a feature matrix
- inherit the old homepage section logic just because it exists

### Hero answer
- hero burden:
  - atmosphere
  - category stamp
  - one emotional transformation beat
- hero non-burden:
  - detailed feature explanation
  - trust-chain proof
  - commercial argument

### Motion answer
- motion should express:
  - breath
  - emergence
  - continuity
  - quiet intelligence
- motion should not express:
  - urgency
  - spectacle for its own sake
  - constant novelty

### CTA answer
- the homepage should create two exits only:
  - learn the system
  - enter the commercial path
- it should not ask for too many directions at once

### Copy answer
- shortest copy in the system
- strongest discipline in the system
- the page should feel like a sequence of inevitabilities, not explanation blocks

### Asset answer
- hero family should likely come from `organism`
- supporting sections should lean `state-flow` and `vessel`
- `mindform` should be used sparingly and with intent on the homepage

### Pricing answer
- pricing should feel like a continuation of the site, not a sales tool bolted on
- it should clarify the offer, not widen the story

### Enter answer
- enter should feel like a calm threshold
- it should not overstate auth or entitlement truth
- it should not carry homepage-scale emotional ambition
