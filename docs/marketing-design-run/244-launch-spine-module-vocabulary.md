# 244 Launch Spine Module Vocabulary

Purpose: define the reusable module language for the Launch Spine so the homepage, pricing, and enter surfaces are built from a coherent system instead of page-local invention.

## Core rule
Launch Spine modules must feel like one calm intelligence making contact with a human under load. Modules exist to lower cognitive burden, clarify the next move, and preserve atmosphere.

## Module classes

### 1. Category Lockup
Job:
- establish category, arrival, and quiet authority in one glance

Contains:
- category stamp
- emotional punchline
- restrained proof/support line

Used on:
- homepage hero
- launch-surface headers where category restatement is required

Must not:
- become a campaign slogan stack
- carry more than one proof line
- compete with the hero asset

### 2. Bridge Pause
Job:
- create chapter transitions without feeling like cards or banners

Contains:
- eyebrow/system label
- punchline
- accent/support line

Used on:
- homepage between major family sections

Must not:
- become a feature callout
- introduce new visual grammar per bridge
- carry operational detail

### 3. Family Window
Job:
- introduce one family truth through one dominant mechanic

Contains:
- family label
- headline
- compressed support line
- one interactive or motion-led field

Used on:
- homepage sections
- future launch-surface explainers

Must not:
- explain the whole product
- carry secondary product families inside the same frame
- rely on dense copy to make the mechanic legible

### 4. Trust Proof Band
Job:
- move the visitor from intrigue to belief with grounded evidence

Contains:
- proof statement
- one evidence abstraction
- one implication line

Used on:
- homepage lower spine
- proof and authority teaser zones

Must not:
- feel like compliance documentation
- dump statistics without narrative frame
- overclaim unsupported outcomes

### 5. Commercial Decision Block
Job:
- make the next commercial move obvious without sales pressure

Contains:
- plan or entry proposition
- what happens next
- calm CTA

Used on:
- pricing
- enter
- CTA zones on `/product` and `/method`

Must not:
- look like aggressive conversion UI
- imply auth/provider states that are not real
- fragment into too many options

### 6. Sanctuary Detail Panel
Job:
- hold supporting detail without breaking the calm surface

Contains:
- short heading
- one or two lines of explanation
- optional stat or supporting cue

Used on:
- pricing/enter support blocks
- secondary support zones on launch routes

Must not:
- become a dashboard tile
- outshine the dominant module on the page
- carry expert-depth content as primary reading experience

## Visual law
Across all Launch Spine modules:
- one unmistakable surface
- dark mineral base
- living light, not decorative glow
- glass only as restraint and containment
- transitions should feel morphed, not stacked

## Motion law
- breath before flourish
- continuity before spectacle
- no snapping UI or hard cuts
- movement should clarify hierarchy, not announce itself

## Content law
- consumer-first
- one page job at a time
- one module burden at a time
- no expert-runtime logic in the primary reading layer
