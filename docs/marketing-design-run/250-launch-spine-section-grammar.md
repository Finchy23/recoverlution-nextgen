# 250 Launch Spine Section Grammar

Purpose: define the section-level grammar for the homepage, pricing, and enter surfaces.

## Homepage grammar
1. Category arrival
- one Category Lockup
- one dominant asset field
- one calm CTA proposition

2. Orientation
- one Bridge Pause
- one Family Window carrying the first system truth

3. Continuity / system intelligence
- second Family Window
- optional third Family Window if it adds a new burden, not repetition

4. Trust turn
- one Trust Proof Band
- one lower CTA/decision posture

5. Entry handoff
- one Commercial Decision Block or calm CTA close

## Pricing grammar
1. Family Thesis Header
2. decision framing
3. plan/commercial block(s)
4. support detail panels
5. quiet proof and entry expectation

## Enter grammar
1. Family Thesis Header
2. state/continuity frame
3. next-action block
4. support detail/guardrail panel
5. quiet reassurance close

## Grammar rules
- each section must do one job
- modules should feel breathed, not stacked
- Bridge Pause only appears where chaptering is actually needed
- no feature-grid fallback
- no dashboard-style summary sections
