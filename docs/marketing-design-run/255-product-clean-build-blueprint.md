# 255 Product Clean-Build Blueprint

Purpose: define the clean-build composition for `/product`.

## Page job
`/product` must explain the organism.

It must answer:
- what are the rooms
- how do they relate
- why this is one system, not a feature inventory

It must not:
- become the method page
- become the proof page
- become the authority page

## Composition sequence

### 1. Thesis Header
Modules:
- Family Thesis Header

Burden:
- state that this is one ecosystem / one operating intelligence

### 2. Whole-system frame
Modules:
- Structured Argument Block

Burden:
- establish the end of fragmentation and the one-system logic

### 3. Room architecture reveal
Modules:
- Architecture Reveal

Burden:
- show the relationship between Universal Player, RecoveryOS, and Interventions

### 4. Family map
Modules:
- Structured Argument Block

Burden:
- introduce the major rooms without collapsing into a feature grid

### 5. Integration turn
Modules:
- Evidence Lens or Architecture Reveal

Burden:
- show how one action travels across the system

### 6. Entry handoff
Modules:
- Gentle Conversion Block

Burden:
- move the reader toward `/method`, `/proof`, or `/enter`

## Asset posture
- primary families: `organism`, `vessel`
- supporting family: `state-flow`
- avoid overusing `mindform` unless it supports a specific argument block

## Copy posture
- articulate, not encyclopedic
- explanation through relation, not inventory
- every section should reduce fragmentation, not add it
