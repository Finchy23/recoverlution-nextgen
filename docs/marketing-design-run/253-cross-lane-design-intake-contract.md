# 253 Cross-Lane Design Intake Contract

Purpose: define the minimum package other lanes must return so the public-site clean-build run can move without drift.

## Design lane must return
- route or family being answered
- module map used
- keep/change/remove decisions
- explicit burden alignment notes
- open doctrine dependencies
- Figma nodes or equivalent review references

## Asset lane must return
- family
- approved/candidate state
- intended surface role
- poster/fallback pair
- crop notes
- do-not-use surfaces

## Copy lane must return
- route/family target
- copy envelope by module
- primary candidate lines
- unresolved doctrine flags
- explicit statements of what not to claim

## Frontend lane must return
- module/component mapping
- what is package authority vs assembly
- route implementation seams
- evidence capture plan
- blockers requiring harmony escalation

## Rule
If any lane returns surface material without route/family/burden alignment, marketing should treat it as reference, not implementation truth.
