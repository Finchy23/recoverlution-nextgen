# 256 Method Clean-Build Blueprint

Purpose: define the clean-build composition for `/method`.

## Page job
`/method` must explain how change happens in this system.

It must answer:
- why state comes before story
- what the loop of change is
- how proof and lived repetition create a new default

It must not:
- become a generalized self-help lesson
- become a product inventory page
- become a clinical whitepaper

## Composition sequence

### 1. Thesis Header
Modules:
- Family Thesis Header

Burden:
- state the governing method truth clearly

### 2. State-before-story argument
Modules:
- Structured Argument Block

Burden:
- explain why the old lecture model fails

### 3. Method reveal
Modules:
- Architecture Reveal

Burden:
- show the loop: sensing, delivery, intervention, receipt, identity update

### 4. Repetition / proof turn
Modules:
- Structured Argument Block
- Evidence Lens (supporting)

Burden:
- explain why proof matters more than insight alone

### 5. Route into deeper trust
Modules:
- Gentle Conversion Block

Burden:
- move the reader into `/proof`, `/authority`, or `/enter`

## Asset posture
- primary families: `state-flow`, `mindform`
- supporting family: `vessel`
- avoid using hero-scale organism logic unless it is serving the loop explanation directly

## Copy posture
- calm, exact, embodied
- no guru tone
- no overclaiming scientific certainty
- every section should help the reader feel the method, not just understand it intellectually
