## Core Decision
The public site will be rebuilt as one coherent operating surface.

It will not:
- inherit the old homepage composition as doctrine
- let hero treatment define the whole system
- let trust pages collapse into expert shells
- let visual ambition outrun calm authority

## Surface Doctrine

### 1. One calm intelligence
- every route must feel authored by the same mind
- every section must feel like it belongs to the same organism
- no page is allowed to become its own campaign

### 2. The homepage is the entrance, not the encyclopedia
- category first
- feeling second
- mechanism third
- proof deferred into the Trust Chain

### 3. The public site works in families
- `Launch Spine`
  - `/`
  - `/pricing`
  - `/enter`
- `Trust Chain`
  - `/product`
  - `/method`
  - `/proof`
  - `/authority`
- `Support Depth`
  - `/science`
  - `/clinical`
  - `/architecture`
  - `/research`
  - `/claims`
- `Transition`
  - `/auth/start`
  - `/auth/callback`
  - `/bind`

### 4. Family behavior rules
- Launch Spine:
  - atmospheric
  - emotionally loaded
  - commercially guided
- Trust Chain:
  - structured
  - legible
  - trust-building
- Support Depth:
  - quieter
  - slower
  - evidence-led
- Transition:
  - minimal
  - procedural
  - reassuring

## Design Resolutions

### Hero
- hero is a system event, not a banner
- one category stamp
- one emotional punch
- one explanatory support line
- motion is allowed only if the still fallback still works

### Bridges
- bridges are breath markers, not decorative separators
- one class, many placements
- they should reset attention, not become mini-campaigns

### Sections
- each section gets one dominant idea
- each section gets one dominant support mechanic
- if a section needs multiple ideas, it belongs on a deeper page

### Copy
- public site copy must read like inevitability, not explanation
- expert language is support depth only
- character envelopes are real constraints, not suggestions

### Motion
- motion must clarify hierarchy, depth, or state
- no motion that exists only to impress
- reduced-motion mode must preserve meaning, not just disable polish

### Assets
- assets must be family-governed
- no route chooses media ad hoc
- hero media is conditional until doctrine harmonizes
- every motion asset needs poster + still fallback + use constraints

## Creative Tooling Doctrine
- Figma is for calibration, alignment, and review
- repo docs and package authority remain the source of truth for implementation
- asset, copy, and design lanes feed the system through defined envelopes and handoff specs
- frontend assembles; it does not invent doctrine locally

## Review-Ready Standard
The public site is review-ready only when:
- Launch Spine feels like one calm intelligence
- Trust Chain feels like one coherent trust sequence
- Support Depth no longer reads like expert-first fallback
- Transition remains calm and truthful
- media, motion, and copy all obey the same design law
