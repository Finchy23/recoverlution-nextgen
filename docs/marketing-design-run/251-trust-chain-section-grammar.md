# 251 Trust Chain Section Grammar

Purpose: define the section grammar for `/product`, `/method`, `/proof`, and `/authority` so Trust Chain pages are coherent and non-redundant.

## Shared page flow
1. thesis
- Family Thesis Header

2. first argument move
- Structured Argument Block

3. hidden system reveal or evidence turn
- Architecture Reveal or Evidence Lens

4. second argument move
- Structured Argument Block

5. conversion or continuity handoff
- Gentle Conversion Block

## Route-specific emphasis

### `/product`
- emphasize organism and room-to-room coherence
- use Architecture Reveal earlier
- subordinate Evidence Lens

### `/method`
- emphasize state before story and the loop of change
- use Architecture Reveal to show how body/proof/repetition connect
- subordinate Evidence Lens

### `/proof`
- emphasize Evidence Lens earliest and most heavily
- keep architecture secondary

### `/authority`
- emphasize Structured Argument Block and Evidence Lens
- architecture only where it supports the authority claim

## Section rules
- do not let any Trust Chain page become a long manifesto
- do not let expert runtime language dominate body copy
- do not repeat the homepage hero burden
- each page must feel like one step deeper into belief, not a restart
