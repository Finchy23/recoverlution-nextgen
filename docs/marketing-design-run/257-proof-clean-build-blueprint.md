# 257 Proof Clean-Build Blueprint

Purpose: define the clean-build composition for `/proof`.

## Page job
`/proof` must explain what counts as evidence here and why it matters.

It must answer:
- what a micro-receipt is
- why proof is different from hope
- how one action creates three meanings

## Composition sequence
1. Thesis Header
2. Evidence Lens
3. Structured Argument Block
4. Second Evidence Lens or supporting Architecture Reveal
5. Gentle Conversion Block

## Asset posture
- primary families: `mindform`, `state-flow`
- use assets sparingly and only to support the evidence frame

## Copy posture
- grounded, non-salesy
- proof over hype
- no dashboard rhetoric
