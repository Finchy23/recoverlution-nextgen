## Purpose
- stop treating the current homepage as the design baseline
- interrogate the clean-build homepage as a full-system design problem
- define the questions that must be answered before hero, bridge, section, copy, asset, and motion decisions harden

## Working Rule
- the current homepage is reference only
- the route spine stays fixed
- the homepage must be designed as the public face of the whole organism, not as a prettier version of the old shell
- no single asset, section, or slogan is allowed to define the system by accident

## Question Set

### 1. Arrival questions
- What must a first-time visitor feel in the first `3 seconds`?
- What must they understand in the first `10 seconds`?
- What must they want to do by the first scroll break?
- What should the page refuse to explain on first contact?

### 2. Category questions
- What is the shortest category claim the homepage must establish?
- What is the difference between `Recoverlution` and `RecoveryOS` on the glass?
- Where does the category stamp live, and where does it stop?
- What is internal doctrine only, and what is public category language?

### 3. Hero questions
- What is the hero proving: category, feeling, mechanism, or inevitability?
- What does the hero asset need to carry that typography should not?
- What remains true if the hero runs as a still, not motion?
- What happens if the visitor never watches the hero for more than `2 seconds`?

### 4. Motion questions
- What motions are structural, not decorative?
- What does the page do when motion is reduced or unavailable?
- Where should morph, drift, gravity, convergence, and release show up?
- Which transitions create trust, and which transitions create performance theatre?

### 5. Trust questions
- Where does the page stop being atmospheric and start becoming legible?
- How does trust accumulate without dropping into dashboard language?
- Which sections are proving the system, and which are only orienting the reader?
- How much proof belongs on the homepage versus the trust-chain pages?

### 6. Architecture questions
- What is the minimum section grammar that can carry the whole site?
- Which modules are universal across Launch Spine and Trust Chain?
- Which modules belong only to the homepage?
- Which current modules should be retired instead of refined?

### 7. Asset questions
- What asset families are required for the homepage to function as a system?
- Which family owns the hero?
- Which families clarify the argument versus merely decorate it?
- Which surfaces require motion, and which should stay still?

### 8. Copy questions
- What words must the homepage own?
- What words must be banned from first-screen public language?
- How many characters can each section carry before the page feels like explanation instead of inevitability?
- What must be deferred to `/product`, `/method`, `/proof`, and `/authority`?

### 9. Commercial questions
- What does the homepage owe `/pricing` and `/enter`?
- How does it create desire without feeling like a checkout funnel?
- What must be true before a visitor reaches the first CTA?
- How do we keep the commercial path calm even when the page becomes more ambitious?

### 10. Questions most teams do not ask
- What does the page feel like when it becomes quieter?
- What does the page feel like when the hero asset fails?
- What remains if the copy gets cut in half?
- What should the page never ask the visitor to do?
- What is the first thing we remove if the page starts trying too hard?

## Provisional Answers

### What the homepage is for
- establish category
- create biological calm
- reveal one unmistakable intelligence
- make the system desirable
- hand off trust-building to deeper pages without feeling incomplete

### What the homepage is not for
- explaining every modality
- acting as the main proof page
- carrying expert language on first screen
- forcing the old homepage sections to survive as doctrine

### What the hero must do
- make the page feel already-arrived
- carry one emotional truth and one category truth
- establish atmosphere, not feature inventory
- prove that the surface is alive without becoming busy

### What the page should optimize for
- inevitability
- coherence
- emotional clarity
- calm authority
- one unmistakable surface

## Output Required From This Interrogation
- one homepage clean-build doctrine
- one system-design answer set for hero, bridges, sections, motion, copy, and assets
- one Linear-ready run for Launch Spine execution
