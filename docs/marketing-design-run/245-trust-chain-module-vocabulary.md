# 245 Trust Chain Module Vocabulary

Purpose: define the reusable module language for `/product`, `/method`, `/proof`, and `/authority` so they read as one trust-building family rather than four unrelated long-form pages.

## Core rule
Trust Chain modules must convert category intrigue into structured belief. They should feel calmer, denser, and more articulate than the homepage while remaining human-first and non-clinical in their primary reading layer.

## Module classes

### 1. Family Thesis Header
Job:
- state the page truth immediately

Contains:
- page eyebrow
- thesis headline
- compressed thesis support line

Used on:
- top of all Trust Chain pages

Must not:
- repeat homepage hero posture
- over-explain the whole system
- open with expert-depth framing

### 2. Structured Argument Block
Job:
- advance one step in the trust argument

Contains:
- argument heading
- body copy
- one support cue or visual motif

Used on:
- all Trust Chain pages

Must not:
- collapse into essay formatting
- carry more than one argumentative move
- rely on unsupported claims

### 3. Evidence Lens
Job:
- show proof without turning into a report

Contains:
- evidence frame
- one signal, case pattern, or systems readout
- one interpretation line

Used on:
- `/proof`
- `/authority`
- secondary support zones in `/product` and `/method`

Must not:
- use dashboard rhetoric
- imply finished product telemetry where only doctrine exists
- require product literacy to understand the point

### 4. Architecture Reveal
Job:
- expose hidden system intelligence without overwhelming the reader

Contains:
- concept label
- short explanatory stack
- optional diagrammatic or asset support

Used on:
- `/product`
- `/method`
- `/authority`
- `/architecture`

Must not:
- expose implementation complexity for its own sake
- read like internal documentation
- break the Architecture of Grace rule

### 5. Gentle Conversion Block
Job:
- connect belief to next action without becoming sales copy

Contains:
- page-local conversion proposition
- calm entry explanation
- CTA

Used on:
- lower zones of Trust Chain pages

Must not:
- pressure the visitor
- over-promise outcome
- conflict with current auth/commercial truth

## Family texture
- more editorial than Launch Spine
- more structured than Support Depth
- still visibly part of the same organism
- fewer decorative effects, more measured conviction

## Content law
- thesis first
- architecture second
- evidence third
- CTA last
- expert depth can support but must not dominate the reading experience
