# 254 Homepage Clean-Build Blueprint

Purpose: define the clean-build homepage composition against Launch Spine doctrine, without inheriting the current homepage layout as truth.

## Page job
The homepage must:
- establish category
- create biological contact
- show that this is one calm intelligence, not a bundle of features
- create enough trust to justify the next move

The homepage must not:
- explain every modality
- prove every clinical claim
- act as the whole site in one page

## Composition sequence

### 1. Arrival
Modules:
- Category Lockup
- dominant hero asset field
- calm primary CTA

Burden:
- category arrival
- emotional recognition
- unmistakable surface

### 2. Orientation Bridge
Modules:
- Bridge Pause

Burden:
- chapter the move from awe into understanding

### 3. System Window 1
Modules:
- Family Window

Burden:
- explain the first major truth of the system with one mechanic

### 4. System Window 2
Modules:
- Family Window

Burden:
- show that the intelligence is orchestrated, not static

### 5. System Window 3
Modules:
- Family Window (only if it adds a new burden)

Burden:
- prove that there is real architecture underneath the calm

### 6. Trust Turn
Modules:
- Bridge Pause
- Trust Proof Band

Burden:
- move the visitor from intrigue to belief

### 7. Entry Continuity
Modules:
- Commercial Decision Block
- optional Sanctuary Detail Panel

Burden:
- make the next move calm, honest, and easy to understand

## Asset posture
- hero should be `organism` or another family that can carry category arrival without literal explanation
- supporting sections should not all use the same asset family
- assets must follow burden, not decoration

## Copy posture
- less manifesto, more inevitability
- category and emotional burden first
- supporting proof lines only where needed
- no feature-grid copy

## Motion posture
- arrival should breathe, not announce itself
- section transitions should feel morphed, not card-stacked
- motion should clarify chaptering and intelligence, not create spectacle
