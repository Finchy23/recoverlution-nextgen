## Purpose
- give the lanes a hard filter for whether a public-site decision belongs in the build

## Keep if
- it strengthens one calm intelligence
- it clarifies family behavior
- it reduces cognitive burden
- it survives as still structure, not just motion
- it improves route-to-route coherence
- it makes the commercial path calmer or clearer

## Change if
- it is right in spirit but overworked in form
- it belongs to the right family but is carrying too much burden
- it depends on unresolved hero/media doctrine
- it makes the page feel more impressive than inevitable

## Remove if
- it only exists because it was in the old homepage
- it duplicates a page job handled better elsewhere
- it introduces a new aesthetic sub-brand
- it creates dashboard or feature-grid posture
- removing it makes the page clearer

## Escalate if
- it changes naming hierarchy
- it changes hero/media doctrine
- it introduces unsupported product truth
- it depends on unresolved auth/commercial assumptions
- it changes what a route is for
