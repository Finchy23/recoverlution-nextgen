# Engineering Directive

## Source of Truth (Current)
- Runtime and lab: `Command Center Execution Plan/`
- Design tokens and mechanics are authoritative there until extracted.

## Build Rules
1. Never fork token systems locally in feature modules.
2. Use shared mechanics and stage models.
3. Keep copy and UX consistent with anti-shame + state-first principles.
4. No promise language implying instant cures or singular-cue outcomes.
5. Validate with drift/token guards before shipping.

## Frontend Direction
- Atmosphere/background/color must map to context and safety.
- Hero innovation focus is entrance and interaction component diversity.
- Keep zero-cognitive-load behavior and behind-the-curtain framing.

## Backend Direction
- Supabase as spine for users, cues, journeys, receipts, and governance.
- Notifications/comms are event-driven from user state + consent rules.

## Delivery Sequence
1. Marketing completion
2. Auth shell
3. Module-by-module rollout
4. Mobile parity
