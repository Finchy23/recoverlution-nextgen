## What to improve next

### 1. Prompt ceremony

The current lift is directionally right, but it still wants more grace.

Improve:

- opener weight and placement
- preamble restraint
- vertical spine / breathing thread feel
- spacing between prompt seeds
- the feeling that the prompt is completing the user's thought

Do not turn it into cards, pills, or menu UI.

### 2. Branch ceremony

This is the biggest missing WIP4 seam.

We want the next pass to design how:

- the first selected prompt deepens
- one or two next prompt branches appear
- the branch choice narrows without feeling decision-tree mechanical

This should feel like thought getting truer, not UI getting denser.

### 3. Writing page

The writing surface still needs more emotional authority.

Improve:

- page proportions
- prompt-to-page transition
- page quietness
- text-entry hierarchy
- seal-led gravity

The page must feel like a writing surface, not a form field inside a prototype.

### 4. Gateway

The `write` / `speak` gateway is present, but still too early-stage.

Improve:

- timeless iconography
- vessel distinction
- reverence of the threshold
- calmness of the choice

The gateway should feel like:

- pen as a private letter
- voice as a witnessed utterance

It should not feel like:

- toggle buttons
- feature options
- app controls

### 5. Mobile truth

We need one clear answer for mobile:

- how the prompt ceremony compresses
- how the page holds priority
- how the gateway fits without crowding
- how thread/bridge stay subordinate

Do not solve desktop first and let mobile collapse afterward.

## What not to change

- analytics contracts

- Supabase transport logic
